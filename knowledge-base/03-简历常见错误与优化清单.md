# 简历常见错误与优化清单

## 一、致命错误（一票否决）

### 1. 虚假信息
- ❌ 夸大项目规模（日均千万级实际只有十万级）
- ❌ 把团队成果写成个人成果
- ❌ 伪造学历、工作经历

**后果**: 背调不通过，进入公司黑名单，行业口碑崩塌。

### 2. 简历与岗位完全不匹配
- ❌ 应聘 Java 后端但简历全是前端项目
- ❌ 技术栈与 JD 完全无交集
- ❌ 应聘高级岗位但项目深度不够

**优化**: 针对每个岗位调整项目描述的侧重点，突出匹配的技术关键词。

### 3. 大段空白/频繁跳槽
- ❌ 每段工作经历不足 1年
- ❌ 有超过 3个月的空白期且无合理解释

**优化**: 工作经历按实际情况写，空白期可写"自由职业/技术学习/个人项目"。

## 二、高频错误（严重影响通过率）

### 4. 项目描述只有职责没有成果

**错误示范**:
> 负责订单系统的开发与维护，使用 Spring Boot + MyBatis + MySQL。

**正确示范**:
> 设计并实现订单核心服务，基于 Spring Boot + MyBatis 构建订单履约流程，引入 Redis 缓存热点商品库存信息，将订单创建接口 P99 响应时间从 500ms 降至 50ms，支撑日均 30万+ 订单量。

### 5. 技术栈堆砌

**错误示范**:
> 熟悉 Java, Python, Go, C++, JavaScript, TypeScript, Rust, Kotlin, Scala, Ruby, PHP...

**正确示范**:
> 精通 Java（5年），深入理解 JVM 原理与并发编程；熟悉 Python，能独立完成数据处理脚本开发；了解 Go，阅读过部分开源项目源码。

### 6. 项目经历过于简略

**错误示范**:
> 参与电商系统开发，负责后端接口开发。

**正确示范**:
> 作为核心开发参与 B2C 电商交易系统（DAU 80万+），独立负责商品中心与购物车模块。设计商品多级分类索引方案（ES + MySQL），实现购物车合并策略，支持多端同步。商品搜索接口 QPS 达到 5000+，搜索响应时间 < 100ms。

### 7. 照搬 JD 关键词

**错误**: 简历中满是"熟悉分布式系统"但项目经历中没有任何分布式相关的描述。

**优化**: 每一个写在"技能"栏的技术，必须在项目经历中有对应的实战体现。

### 8. 排版混乱

- ❌ 字体大小不一、对齐混乱
- ❌ 使用花哨的模板（颜色过多、图表过多）
- ❌ 简历超过 3 页
- ❌ 导出为图片格式（ATS 系统无法解析）

**建议**: 简洁黑白模板，PDF 格式，1-2页。

## 三、易忽视错误（影响面试官印象）

### 9. 邮箱/手机号错误

投递前务必确认联系方式正确，建议使用专业邮箱（如 Gmail、Outlook）。

### 10. 文件命名不规范

- ❌ `简历最终版v3(1)(修改).pdf`
- ✅ `张三_Java后端开发_5年经验.pdf`

### 11. 没有针对性调整

用同一份简历投递所有岗位，没有根据 JD 调整项目描述的侧重点。

**优化**: 至少调整以下几点：
- 项目描述中突出与 JD 匹配的技术关键词
- 调整项目顺序（最匹配的放第一个）
- 技能清单中突出岗位需要的技能

### 12. 缺少技术亮点

所有项目都是"增删改查"，没有技术难点和亮点，面试官会觉得深度不够。

**优化**: 每个项目至少保留 1 个技术亮点：
- 性能优化（数据库/接口/缓存）
- 架构演进（重构/微服务化）
- 技术攻坚（分布式事务/高并发/数据一致性）
- 工程化（CI/CD/自动化测试/监控）

## 四、快速自查清单

投递前逐项检查：

### 基础信息
- [ ] 联系方式正确（手机号、邮箱）
- [ ] 文件名为 `姓名_岗位_经验年限.pdf`
- [ ] 简历为 PDF 格式，不超过 2 页（初级 1 页）
- [ ] 排版整洁，无错别字

### 项目经历
- [ ] 每个项目有 S/T/A/R 完整描述
- [ ] 至少 70% 的描述有量化数据
- [ ] 使用强动词开头，避免"负责""参与"
- [ ] 每个项目至少 1 个技术亮点
- [ ] 项目描述与技能清单一致（不矛盾）

### 技能清单
- [ ] 掌握程度用词准确（精通/深入理解/熟练掌握/了解）
- [ ] 列出的技术都有项目经历支撑
- [ ] 技术关键词与目标岗位 JD 匹配

### 针对性
- [ ] 已根据目标 JD 调整项目描述侧重点
- [ ] 最匹配的项目排在最前面
- [ ] 自我评价与岗位要求契合

### 常见扣分项
- [ ] 没有"精通XX"却到处写精通
- [ ] 没有把团队成果写成个人成果
- [ ] 没有工作经历断层超过 3 个月
- [ ] 没有频繁跳槽（每段 < 1年）
