# 大厂JD关键词与能力模型

## 一、阿里职级体系与能力要求

### P 序列（技术岗）

| 职级 | 对应年限 | 核心能力要求 | 典型 JD 关键词 |
|------|---------|-------------|---------------|
| P5 | 应届/1年 | 能独立完成模块开发 | Java基础、Spring Boot、MySQL、编码规范 |
| P6 | 2-5年 | 能独立负责子系统设计 | 微服务、分布式系统、性能优化、领域设计 |
| P7 | 5-8年 | 能负责系统级架构设计 | 架构设计、技术选型、高可用、团队指导 |
| P8 | 8年+ | 能负责业务域技术规划 | 技术战略、跨域架构、技术影响力、团队建设 |

### 阿里面试官视角

- **看重**: 技术深度（原理级理解）、业务理解力、系统性思维、拥抱变化
- **反感**: 只会用框架不理解原理、没有业务思考、技术方案缺乏 trade-off 分析
- **经典问题**: "你做这个技术选型的理由是什么？""如果流量再大10倍你怎么处理？"

## 二、腾讯职级体系与能力要求

### T 序列（技术岗）

| 职级 | 对应年限 | 核心能力要求 | 典型 JD 关键词 |
|------|---------|-------------|---------------|
| T9 (2-1) | 1-3年 | 独立完成功能模块 | C++/Go/Java、Linux、数据结构、算法 |
| T10 (2-2) | 2-5年 | 独立负责子系统 | 高并发、网络编程、系统设计、性能调优 |
| T11 (3-1) | 5-8年 | 系统架构设计 | 分布式架构、海量数据处理、技术方案评审 |
| T12 (3-2) | 8年+ | 技术方向规划 | 技术架构演进、跨团队协作、技术梯队培养 |

### 腾讯面试官视角

- **看重**: 编码能力（算法手撕）、系统设计能力、工程素养、产品质量意识
- **反感**: 代码不规范、不考虑边界条件、缺乏线上敬畏心
- **经典问题**: 算法题（必考）、"设计一个短链系统"、"如何保证消息不丢不重"

- **经典问题**: "你的方案 AB 测试数据是什么？""这个功能的核心指标是什么？"

## 四、JD 关键词对照表

### 按能力维度

| JD 关键词 | 实际考察点 | 简历应体现 |
|-----------|-----------|-----------|
| "高并发/大流量" | 限流、缓存、异步、分库分表 | 具体并发量数据 + 技术方案 |
| "高可用" | 熔断降级、容灾、监控告警 | 可用性数据(99.99%) + 方案 |
| "分布式系统" | 分布式事务、一致性、CAP | 具体场景 + 选型理由 |
| "性能优化" | 数据库优化、JVM 调优、慢接口 | 优化前后的量化对比 |
| "技术架构" | 系统设计、技术选型、演进 | 架构图 + 演进路径 + trade-off |
| "海量数据" | 分库分表、ES、离线计算 | 数据量级 + 处理方案 |
| "微服务" | 服务拆分、治理、链路追踪 | 服务数量 + 治理方案 |
| "DevOps" | CI/CD、容器化、自动化 | 部署频率 + 自动化覆盖率 |

### 按行为动词

| JD 动词 | 期望看到的内容 |
|----------|--------------|
| "主导" | 你是核心决策者，不是参与者 |
| "设计" | 有架构图/方案文档，且阐述选型理由 |
| "优化" | 优化前后数据对比，方法论总结 |
| "推动" | 跨团队协调能力，推动了什么改变 |
| "建立" | 从0到1搭建了什么体系/规范/工具 |
| "保障" | SLO/SLA 指标，监控告警体系 |

## 五、投递策略

### 1. 简历与 JD 匹配度优化

**步骤**:
1. 提取 JD 中的技术关键词（硬技能）
2. 提取 JD 中的行为关键词（软技能/能力要求）
3. 对比自己简历，标注匹配项和缺失项
4. 在项目经历中补充/强化匹配项的描述
5. 调整项目顺序（最匹配的放前面）

### 2. 不同级别简历侧重点

| 应聘级别 | 简历核心 | 项目描述重点 |
|----------|---------|-------------|
| 初级(1-3年) | 基础扎实 + 快速学习 | 具体做了什么 + 技术细节 |
| 中级(3-5年) | 独立负责 + 技术深度 | 技术方案 + 量化结果 |
| 高级(5-8年) | 架构能力 + 技术视野 | 系统设计 + trade-off + 业务价值 |
| 资深/专家 | 技术影响力 + 团队建设 | 技术战略 + 团队成果 + 行业贡献 |

### 3. 大厂简历加分项

- **开源贡献**: GitHub PR/Issue、开源项目维护
- **技术博客**: 掘金/CSDN/个人博客的技术文章
- **技术演讲**: QCon/ArchSummit/公司内部分享
- **专利/论文**: 发明专利、学术论文
- **竞赛**: ACM/LeetCode/黑客马拉松获奖
- **技术社区**: Stack Overflow 问答、技术群答疑
