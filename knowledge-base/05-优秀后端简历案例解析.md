**项目背景**: 公司核心电商交易平台，日均 GMV 3000万+，DAU 80万+，原有单体架构在促销期间频繁超时，需进行微服务化改造。

#### 项目一：实时风控引擎

**时间**: 2023.06 - 至今
**角色**: 技术负责人
**技术栈**: Go 1.21 + gRPC + Kafka + Redis + ClickHouse + K8s

**项目背景**: 公司金融业务线需要实时风控系统，要求在 50ms 内完成单笔交易的风险评估，日处理交易 2000万+。

**核心工作**:
- **架构设计**: 设计基于规则引擎 + ML 模型的混合风控架构，规则引擎负责实时拦截（P99 < 20ms），ML 模型负责异步评估（P99 < 200ms）。系统采用 Go 实现核心计算层，通过 gRPC 对外提供服务
- **实现规则引擎**: 开发基于 DAG（有向无环图）的规则编排引擎，支持 200+ 条风控规则的动态配置与热更新，单次评估耗时稳定在 15ms 以内
- **建设可观测性**: 基于 OpenTelemetry 实现全链路追踪，ClickHouse 存储风控日志并搭建实时分析大盘，风控事件排查时间从 2小时缩短至 10分钟
- **性能调优**: 通过 pprof 排查并修复 3 处内存泄漏问题，优化 goroutine 池配置，单机吞吐量从 5000 TPS 提升至 20000 TPS

**技术亮点**: DAG 规则引擎 + gRPC 流式通信 + pprof 性能调优实战。
