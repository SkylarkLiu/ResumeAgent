# 智能体工程师岗位画像与技能地图

## 一、岗位概述

智能体工程师（AI Agent Engineer）是 AI 应用层的核心角色，负责将大语言模型能力编排为可完成复杂任务的自主系统。与传统 ML 工程师不同，智能体工程师更侧重于 **LLM 应用的架构设计、Prompt 工程、工作流编排和系统可靠性**。

### 岗位演进

```
传统开发工程师 → ML工程师 → LLM应用工程师 → 智能体工程师
  (CRUD)      (模型训练)   (RAG/微调)     (Agent/工作流编排)
```

## 二、核心能力模型

### 2.1 能力雷达图（权重分布）

| 能力维度 | 权重 | 说明 |
|----------|------|------|
| **LLM 应用开发** | 30% | Prompt 工程、API 调用、输出控制 |
| **Agent 编排** | 25% | LangGraph/CrewAI、工作流设计、状态管理 |
| **RAG 系统** | 20% | 向量检索、分块策略、检索优化、评估 |
| **系统工程** | 15% | API 设计、性能优化、可观测性 |
| **领域理解** | 10% | 业务建模、产品思维、用户场景 |

### 2.2 六级能力阶梯

| 级别 | 名称 | 能力要求 | 典型职责 |
|------|------|----------|----------|
| **L1** | 初级 | 能调用 LLM API，编写基础 Prompt | 开发简单的 LLM 调用脚本 |
| **L2** | 中级 | 能搭建 RAG 系统，理解 Embedding 和向量检索 | 独立完成知识库问答系统 |
| **L3** | 高级 | 能设计多步 Agent 工作流，处理复杂状态 | 设计并实现完整 Agent 应用 |
| **L4** | 资深 | 能优化 RAG 效果，设计可靠的生产系统 | 负责架构设计、性能调优、评估体系 |
| **L5** | 专家 | 能创新 Agent 架构模式，解决前沿问题 | 技术选型、团队指导、技术突破 |
| **L6** | 首席 | 能定义行业标准，推动技术方向 | 技术战略、开源贡献、行业影响力 |

## 四、核心知识体系

### 4.1 Prompt 工程（L1-L3）

```
基础指令 → Few-Shot → Chain-of-Thought → 结构化输出 → DSPy 自动优化
```

| 技能 | 级别 | 关键知识点 |
|------|------|-----------|
| 基础 Prompt 设计 | L1 | 角色设定、指令清晰、格式约束 |
| Few-Shot 示例 | L2 | 示例选择、格式一致性 |
| Chain-of-Thought | L2 | 推理链引导、思维树 |
| 结构化输出 | L2 | JSON Schema、OutputParser、容错解析 |
| Prompt 模板管理 | L3 | 版本管理、A/B 测试、参数化 |
| DSPy | L4 | 自动 Prompt 优化、示例选择器 |

### 4.2 RAG 系统（L2-L4）

```
文档加载 → 分块 → Embedding → 向量存储 → 检索 → 重排 → 上下文拼装 → LLM 生成
```

| 技能 | 级别 | 关键知识点 |
|------|------|-----------|
| 基础 RAG | L2 | 文本分块、向量检索、上下文拼装 |
| 高级检索 | L3 | 查询改写、多路召回、重排序 |
| 评估体系 | L3 | RAGAS、Faithfulness、Answer Relevancy |
| 生产优化 | L4 | 混合检索、增量索引、缓存策略 |
| 大规模部署 | L4 | 分布式向量库、性能调优、监控 |

### 4.3 Agent 编排（L2-L5）

```
单 LLM 调用 → Chain → Router-Agent → 多 Agent 协作 → 自主规划
```

| 技能 | 级别 | 关键知识点 |
|------|------|-----------|
| 基础编排 | L2 | LangChain Chain、顺序执行 |
| 状态图 | L3 | LangGraph StateGraph、条件边、子图 |
| 持久化 | L3 | Checkpointer、多轮对话、状态恢复 |
| 工具调用 | L3 | Function Calling、工具注册、错误处理 |
| 人机协作 | L4 | interrupt 机制、审批流、人工接管 |
| 多 Agent | L4 | Agent 间通信、任务分配、共识机制 |
| 自主规划 | L5 | ReAct、Plan-and-Execute、反思循环 |

## 五、面试高频考点

### 5.1 概念理解类

| 问题 | 考察点 | 参考回答框架 |
|------|--------|-------------|
| RAG 和微调的区别？ | 方案选型能力 | RAG：知识动态更新、成本低、无需训练；微调：风格/格式定制、领域知识内化、成本高 |
| LangGraph 和 LangChain 的区别？ | 框架理解 | LangChain：线性 Chain；LangGraph：图拓扑、循环、状态持久化 |
| 如何解决 LLM 幻觉？ | 工程经验 | RAG 约束、引用标注、Faithfulness 评估、温度降低、Grounding 检查 |
| 如何评估 Agent 系统质量？ | 评估体系 | 检索质量（Recall/MRR）、生成质量（Faithfulness/Relevancy）、端到端指标 |
| Token 消耗如何优化？ | 成本意识 | 上下文裁剪、缓存、模型分级（路由→小模型、生成→大模型）、Prompt 精简 |

### 5.2 系统设计类

| 问题 | 设计要点 |
|------|----------|
| 设计一个企业知识库问答系统 | 文档加载→分块→Embedding→向量库→检索→RAG→流式输出→引用标注→评估 |
| 设计一个多步 Agent 系统 | StateGraph、条件路由、工具注册、状态持久化、错误恢复、人机协作 |
| 如何处理超长文档？ | 分层检索（摘要→段落→原文）、Map-Reduce、Map-Chain、滚动窗口 |
| 如何实现实时搜索增强？ | Tavily/SerpAPI 集成、路由决策、搜索结果标准化、时效性标注 |
| 设计简历分析 Agent | 多模态提取→结构化→JD 检索→对比分析→评分报告→引用来源 |

### 5.3 工程实践类

| 问题 | 考察点 |
|------|--------|
| JSON 结构化输出不稳定怎么办？ | Schema 约束 + 后处理容错 + 重试 + Few-Shot 示例 |
| 向量检索召回率低怎么办？ | 查询改写、多路召回、重排序、分块策略调优、Embedding 模型更换 |
| Agent 响应慢怎么办？ | 异步并发、流式输出、缓存热查询、模型分级、并行工具调用 |
| 如何保证 Agent 系统稳定性？ | 超时控制、重试机制、降级策略、监控告警、日志追踪 |

## 七、学习路径建议

### 阶段一：基础入门（1-2 个月）

1. **LLM API 调用**：用智谱/OpenAI API 完成对话、文本生成
2. **Prompt 工程**：掌握角色设定、Few-Shot、CoT
3. **基础 RAG**：用 LangChain + FAISS 搭建知识库问答
4. **FastAPI**：搭建简单的后端 API

### 阶段二：Agent 开发（2-3 个月）

1. **LangGraph 入门**：StateGraph、条件边、Checkpointer
2. **多步工作流**：Router → 检索 → 生成 的完整 Agent
3. **结构化输出**：JSON 提取、Pydantic Schema
4. **工具调用**：Function Calling、自定义工具

### 阶段三：生产优化（2-4 个月）

1. **RAG 优化**：查询改写、重排序、多路召回、评估
2. **系统可靠性**：超时、重试、降级、监控
3. **性能优化**：异步并发、缓存、流式输出
4. **可观测性**：LangSmith 集成、日志追踪

### 阶段四：进阶方向（持续）

1. **多 Agent 协作**：CrewAI、AutoGen、自研框架
2. **DSPy**：自动化 Prompt 优化
3. **模型评估**：RAGAS、自定义评估指标
4. **前沿跟进**：论文阅读、开源项目贡献
